内部类

定义

  • 如果一个事物的内部包含另一个事物，那么就是一个类包含另一个类。即，内部类。

  • 例如：身体与心脏的关系。

  • 分类：

    • 成员内部类
    • 局部内部类（包含匿名内部类）

定义类时，权限修饰符使用规则：

  1. 外部类：public/(default)
  2. 成员内部类：public/protected/(default)/private
  3. 局部内部类：什么都不能写，它自己有默认的权限：只能被所在方法使用。

局部内部类的final问题

  • 局部内部类，如果希望访问所在方法的局部变量，那么这个局部变量必须是事实上final的。

  • java8后：只要局部变量事实不变，final关键字可以省略。

  • 原因：

    • new出来的对象在堆内存中
    • 局部变量是跟着方法存储在栈内存当中
    • 方法运行结束后立即出栈，局部变量也会随之消失
    • new出来的对象会在堆中持续存在，直到垃圾回收
    • 一旦局部变量随方法出栈，new对象就找不到这个变量，所以作为常量复制到new出的对象中，就可以持续使用。

匿名内部类（重点！！！）

  • 如果接口中的实现类（或者是父类的某个子类）只需要使用唯一的一次，

    那么这种情况下就可以省略该类的定义，改为使用匿名内部类。

  • 定义格式：

    • new代表创建对象的动作；
    • 接口名称代表匿名内部类需要实现哪个接口；
    • {…}才是匿名内部类的内容

  • 示例：

  • 注意事项：

    • 匿名内部类在创建对象的时候，只能使用唯一的一次。

      如果希望多次创建对象，还是定义实现类方便。

    • 匿名内部类对于接口的抽象方法必须全部覆盖重写。

    • 匿名对象在调用方法的时候只能调用唯一一次，如果希望同一个对象调用多次，必须起名。

    • 匿名内部类是省略了实现类/子类，匿名对象是省略了对象名称。

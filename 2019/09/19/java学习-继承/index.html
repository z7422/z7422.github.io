java学习--继承

发表于 2019-09-19 | 分类于 Java学习笔记

一、继承的概述

  • 面向对象的三大特征：封装性、继承性、多态性。

  • 继承是多态的前提，如果没有继承，就没有多态。

  • 继承主要解决的问题是：共性抽取。

  • 父类：被继承的类，也叫做基类、超类。

  • 子类：继承的类，也可以叫做派生类。

  • 继承关系当中的特点：

    1. 子类可以拥有父类的“内容”；
    2. 子类还可以拥有自己专有的内容。

三、继承中成员变量的访问特点

  • 在父子类的继承关系中，如果成员变量重名，则创建子类对象时，访问有两种形式：

    • 直接通过子类对象访问成员变量：等号左边是谁，就优先用谁，没有则向上找。

    • 间接通过成员方法访问成员变量：看方法属于谁，就优先用谁，没有则向上找。

四、区分子类方法中重名的三种变量

局部变量：直接写成员变量名

本类的成员变量：this.成员变量名

父类的成员变量：super.成员变量名

六、方法的重写

  • 重写(override)：在继承关系当中，方法的名称一样，参数列表也一样。

  • 重载与重写的区别：

    • 重写：方法名称一样，参数列表也一样（覆盖/覆写）
    • 重载：方法名称一样，参数列表不一样

  • 特点：创建的是类对象，则优先使用类方法。

  • 注意事项：

    1. 必须保证父子类之间方法的名称相同，参数列表也相同。

    2. @Override：写在方法前面，用来检测是不是有效的正确覆盖重写。//@：注解；不写注解也无所谓

    3. 子类方法的返回值必须小于等于父类方法的返回值范围。

      例如：Object是所有类的公共最高父类，java.lang.String就是Object的子类。

    4. 子类方法的权限必须大于等于父类方法的权限修饰符。

      提示：public > protected > 默认，留空不写 > private

  • 应用场景：对于已经投入使用的类，尽量不要进行修改，推荐定义一个新的类，来重复利用其中共性的内容并且改动添加新的内容。

七、继承中构造方法的访问特点

  • 子类构造方法当中有一个默认隐含的super()调用，所以一定的时先调用父类构造，后调用子类构造方法。

  • 子类构造可以通过super关键字来调用父类重载构造。（当父类重载了有参构造方法，此时父类中不再有系统赠送的无参构造方法，那么子类中构造方法必须调用super(参数)即父类的有参方法来构造，否则会报错。因为父类中并不含有无参构造方法。）

  • super的父类构造调用，必须时子类构造方法的第一个语句。也不能一个子类构造调用多次super构造。

  • 子类必须调用构造方法，不写则赠送super();写了必须用指定的super调用。

八、Super关键字的用法

  1. 在子类的成员方法中，访问父类的成员变量。
  2. 在子类的成员方法中，访问父类的成员方法。
  3. 在子类的构造方法中，访问父类的构造方法。

九、this关键字的用法

  1. 在本类的成员方法中，访问本类的成员变量。

  2. 在本类的成员方法，访问本类的另一个成员方法。

  3. 在本类的构造方法中，访问本类的另一个构造方法。

    在第三个用法中注意：

    ​ 1. 调用另一个构造方法也必须时构造方法的第一个语句。

    ​ 2. super和this两种构造调用也不能同时使用。

十、java继承的三个特点

  1. java语言是单继承的：一个类的直接父类只能有一个。

  2. java语言可以多级继承：A-继承>B-继承>C。C也是A的父类，但不是直接父类。

    java.lang.Object是所有类的父类。

  3. 一个子类的直接父类是唯一的，但是一个父类可以有多个子类。

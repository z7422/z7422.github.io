java学习--继承

发表于 2019-09-19 | 分类于 Java学习笔记

一、继承的概述

  • 面向对象的三大特征：封装性、继承性、多态性。

  • 继承是多态的前提，如果没有继承，就没有多态。

  • 继承主要解决的问题是：共性抽取。

  • 父类：被继承的类，也叫做基类、超类。

  • 子类：继承的类，也可以叫做派生类。

  • 继承关系当中的特点：

    1. 子类可以拥有父类的“内容”；
    2. 子类还可以拥有自己专有的内容。

三、继承中成员变量的访问特点

  • 在父子类的继承关系中，如果成员变量重名，则创建子类对象时，访问有两种形式：

    • 直接通过子类对象访问成员变量：等号左边是谁，就优先用谁，没有则向上找。

    • 间接通过成员方法访问成员变量：看方法属于谁，就优先用谁，没有则向上找。

四、方法的重写

  • 重写(override)：在继承关系当中，方法的名称一样，参数列表也一样。

  • 重载与重写的区别：

    • 重写：方法名称一样，参数列表也一样（覆盖/覆写）
    • 重载：方法名称一样，参数列表不一样

  • 特点：创建的是类对象，则优先使用类方法。

  • 注意事项：

    1. 必须保证父子类之间方法的名称相同，参数列表也相同。

    2. @Override：写在方法前面，用来检测是不是有效的正确覆盖重写。//@：注解；不写注解也无所谓

    3. 子类方法的返回值必须小于等于父类方法的返回值范围。

      例如：Object是所有类的公共最高父类，java.lang.String就是Object的子类。

    4. 子类方法的权限必须大于等于父类方法的权限修饰符。

      提示：public > protected > 默认，留空不写> private

  • 应用场景：对于已经投入使用的类，尽量不要进行修改，推荐定义一个新的类，来重复利用其中共性的内容并且改动添加新的内容。

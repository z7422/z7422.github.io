java学习--多态

发表于 2019-09-24 | 分类于 Java学习笔记

多态Polymorphism

面向对象三大特征：封装性&继承性&多态性

extends继承或者implements实现，是多态性的前提。

多态的格式与使用

  • 代码当中体现多态性：父类引用指向子类对象。(子类被当作父类看待:猫->动物)

  • 多态：java引用变量有两个类型：编译时类型(由声明类型决定)&运行时类型(实际赋值的对象决定)。编译时类型和运行时类型不一致，可能出现多态。

  • 格式：父类名称 对象名 = new 子类名称();

    ​ 接口名称 对象名 = new 实现类名();

  • 注意事项：

    通过引用变量来访问其包含的实例Field时，系统总是试图让问它编译时类型所定义的Field，而不是它运行时类型定义的Filed。

    在使用例如Fu obj = new Zi();语句实现多态时，系统首先在堆内存区放入编译时类型，即Fu对象信息，而在进行赋值操作时，将运行时类型Zi对象信息放入堆内存中。

    多态中成员变量的使用特点

    访问成员变量的两种方式：

    1. 直接通过对象名称访问成员变量：看等号左边是谁，优先用谁，没有则向上找。

    2. 间接通过成员方法访问成员变量：看该方法属于谁，优先用谁，没有则想上找。

多态中成员方法的访问

规则：看new的是谁，就优先用谁，没有则向上找。

注意：编译看左边，运行看右边。

​ (运行)变量看父类，方法看子类。没有向上找。

使用多态的好处

对象的向上转型

向上转型即父类引用指向子类对象，一定是安全的，从小范围转向大范围。

注意：一旦向上转型为父类，那么就无法调用子类原本特有的内容。

对象的向下转型（还原）

  • 向下转型格式：子类名称 对象名 = (子类名称)父类对象；

  • 含义：将父类对象还原成为本来的子类对象。

instanceof关键字进行类型判断

  • 问题：如何才能知道一个父类引用的对象本来是什么子类？

  • 解决： 对象 instanceof 类名称

    ​ 这会的返回一个boolean值的结果，也就是判断前面的对象能不能当作后面类型的实例。

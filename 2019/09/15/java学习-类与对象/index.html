java学习--类与对象

发表于 2019-09-15 | 分类于 Java学习笔记

一、 面向对象思想

  • 面向过程：当需要实现一个功能时，每一个具体的步骤都要亲历亲为，详细处理每一个细节。

  • 面向对象：当需要实现一个功能时，不关心具体的步骤，而是寻找一个具有该功能的工具来实现。

二、类与对象的关系

  • 类:是一组相关属性和行为的集合，可以看成是一类事物的模板，使用事物的属性特征和行为特征来描述该类事物。

    举例：猫 属性：名字、体重、年龄、颜色

    ​ 行为：走、跑、跳、抓老鼠

  • 对象：类是对一类事物的描述（抽象），而对象是一类事物的一个实例（具体）。

  • 关系：对象是类的实体，类是对象的模板。

  • Java源文件命名规则：

    • 通常情况下下，java程序源文件的主文件名可以是任意的。
    • 如果定义了public类，那么该源文件的主文件名必须与该public类名相同。
    • 一个java源文件里最多一个pubic类，可以有多个类。

三、对象的创建及使用

  2. 创建：（格式） 类名称 对象名 = new 类名称();

    ​ Student stu = new Student();

四、一个对象的内存图

​ 如图所示，在程序运行前将class信息存入方法区，从main方法开始执行。创建Phone的对象one时，new Phone()的信息放入堆内存中，包含成员变量及成员方法；成员变量存储默认值直接存放在堆中；成员方法存储的是方法的内存地址值，指向方法区中类的成员方法。对象one在栈中存放的一个内存地址值，指向堆内存。

​ 使用one.brand=“苹果”等语句向成员变量赋值，是通过栈中one指向的地址访问堆内存中的成员变量，对其赋值。使用one.call(“乔布斯”)语句，有两个步骤，首先通过栈内存中的值访问堆内存，再通过堆中的地址访问方法区中的方法；call方法运行时需要进栈（压栈），将参数传递并进行方法调用，调用结束call方法出栈。继续进行one.sendMessage()方法，使之进栈并调用方法，随后出栈。main方法结束，同样出栈，程序停止。

​ Tips：对象调用方法时，根据对象中方法标记（地址值），去类中寻找方法信息。这样哪怕是多个对象，方法信息 只保存一份，节约内存空间

九、局部变量与成员变量的区别

  1. 定义的位置不同：

    局部变量:在方法的内部

    成员变量:在方法的外部，直接写在类当中

  2. 作用范围不同：

    局部变量:只有在方法当中才可以使用，出了方法就不能再用

    成员变量:整个类中都可以使用

  3. 默认值不同：

    局部变量:没有默认值，如果想使用必须手动进行赋值。未赋值进行打印会报错，没赋值不能使用。

    成员变量:如果没有赋值，会有默认值（同数组）

    Tips：方法的参数就是局部变量，但是参数在方法调用时必然会传递过来一个值。

  4. 内存的位置不一样

    局部变量：位于栈内存

    成员变量：位于堆内存

  5. 生命周期不一样

    局部变量：随着方法进栈而诞生，随着方法出栈而消失

    成员变量：随着对象创建而诞生，随着对象被垃圾回收而消失

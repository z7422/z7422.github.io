java学习--权限修饰符

发表于 2019-09-18 | 分类于 Java学习笔记

Static

static静态关键字

  • static:只在类当中保存唯一一份数据，所有本类对象共享同一份。

  • 一旦使用static关键字，这样的内容不再属于对象自己，而是属于类的，凡是本类的对象都共享同一份。

静态static关键字修饰成员变量

  • 如果一个成员变量使用了static关键字，那么这个变量不再属于对象自己，而是属于所在的类，多个对象共享同一份数据。通过任何一个对象修改该成员变量，所有对象中这个值都会发生改变。

静态static关键字修饰成员方法

  • 静态方法不属于对象，而是属于类的。
  • 如果没有static关键字，那么必须先创建对象，然后通过对象调用方法。
  • 有static关键字，即静态方法，可以通过对象名进行调用，也可以通过类名进行调用（推荐）。
  • 对于本类当中的静态方法，可以省略类名称。
  • 注意事项：
    1. 静态不能直接访问非静态。（原因：因为在内存当中是先有的静态内容，后有的非静态内容。）
    2. 静态方法中不能使用this。（原因：this代表当前对象，通过谁调用的方法，谁就是当前对象。

静态static的内存图

  • 方法区中有一块独立的空间叫做静态区，存储静态相关的东西。

  • 根据类名称访问静态成员变量，全程与对象无关，只与类有关。

静态代码块

  • 格式：

    public class 类名称{

    ​ static{

    //静态代码块的内容

    ​ }

    }

  • 特点：当第一次用到本类是，静态代码块执行唯一的一次。

  • 注意：静态内容总是优先于非静态内容。所以静态代码块比构造方法先执行。

  • 静态代码块的典型用途：用来一次性的对静态成员变量进行赋值。

private

private关键字的作用以及使用

  • 问题描述：在定义person对象的年龄时，无法阻止不合理的数值被键入。

    ​ 如:年龄-20

    ​ 解决方案：用private关键字将需要保护的成员变量进行修饰。

    ​ 一旦使用private进行修饰，那么本类当中仍可以随意访问，但是超出本类后就不能直接访问。

    ​ 间接访问private成员变量，就是定义一对儿Getter&Setter方法，提高代码安全性。

final关键字

final关键字代表最终不可改变的。

常见四种用法

  1. 可以用来修饰一个类
  2. 可以用来修饰一个方法
  3. 可以用来修饰一个局部变量
  4. 可以用来修饰一个成员变量

final关键字用来修饰类

  • 格式：

    ​ public final class 类名称{

    ​ //……

    ​ }

  • 含义：当前这个类不能有任何的子类。（太监类）

    ​ 一个类如果是final类，那么其中的成员方法都不能覆盖重写。

final关键字修饰一个方法

  • 格式：

    ​ 修饰符 final void method(){//方法体}

  • 含义：当前这个类不能被覆盖重写。

  • 注意:对于类与方法而言，abstract与final是矛盾的不能同时使用。

final关键字修饰一个局部变量

  • 格式： final int num = 100；
  • 含义:使用final来修饰局部变量，那么这个变量就不能进行更改。
    • 基本类型：变量中的数据不可改变。
    • 引用类型：变量中的地址值不可改变，内容可以变化。

final修饰一个成员变量

  • 含义：对于成员变量而言，使用final关键字，这个变量不可变。
  • 与局部变量的区别：
    1. 成员变量具有默认值，使用了final之后必须手动赋值，不再给默认值。
    2. 一个被final修饰的成员变量，要么直接赋值，要么通过构造方法赋值，二者选其一。

定义类时，权限修饰符使用规则：

  1. 外部类：public/(default)
  2. 成员内部类：public/protected/(default)/private
  3. 局部内部类：什么都不能写，它自己有默认的权限：只能被所在方法使用。

java学习--接口

发表于 2019-09-22 | 分类于 Java学习笔记

接口

一、定义及格式

  • 接口是一种引用数据类型，最重要的内容就是其中的抽象方法。
  • 定义格式：
    • public interface 接口名称{//接口内容}
    • tips：换成了关键字interface之后，编译生成的字节码仍然是：.java–>.class。
  • 接口包含的内容(java7):
    • 常量
    • 抽象方法
    • 默认方法(java8追加)
    • 静态方法(java8追加)
    • 私有方法(java9追加)

二、接口当中的抽象方法

  • 在任何版本的java中，接口都能定义抽象方法：

    ​ public abstract 返回值类型 方法名称(参数列表)；

  • 接口当中的抽象方法，修饰符必须时固定的关键字：public abstract

  • 这两个关键字修饰符，可以选择性地省略。

  • 接口使用步骤：

    • 1.接口不能直接使用，必须有一个实现类来实现该接口。

      格式：public class 实现类名称 implements 接口名称{

      //…

      }

    • 2.接口的实现类必须覆盖重写接口中所有的抽象方法。

    • 3.创建实现类的对象，进行使用。

    • 注意事项：如果实现类也时抽象类，那么可以不全部覆盖重写接口中的所有抽象方法。

三、接口中的默认方法

  • 从Java8开始，接口里允许定义默认方法。

  • 格式：

    • public default 返回值类型 方法名称（参数列表）{方法体}

  • 目的：默认方法可以解决接口升级的问题，不改动之前已经投入使用的实现类（抽象方法必须全部实现，默认方法不用全部实现）。

  • 注意事项：

    1. 接口的默认方法，可以不经过覆盖重写通过接口实现类对象直接调用。
    2. 接口的默认方法，也可以被接口实现类进行覆盖重写。

四、接口中的静态方法

（从java8开始，接口中允许定义静态方法。）

  • 格式： public static 返回值类型 方法名称(参数列表){…方法体}
  • 使用：
    • 不能通过接口实现类的对象来调用接口中的静态方法。因该通过接口名称直接调用其中的静态方法。

五、接口私有方法定义

  • 问题：重复代码很多，我们需要抽取一个共有方法来解决两个默认方法（静态方法）之间重复代码的问题。但是这个公有方法不应该让实现类（用户）使用，应该只让接口中的方法使用，是私有化的。

  • 解决方法：从java9开始，接口当中允许定义私有方法。

    • 普通私有方法：解决多个默认方法之间重复代码的问题。
      • 格式：private 返回值类型 方法名称（参数列表）{方法体}
    • 静态私有方法：解决多个静态方法之间重复代码的问题。
      • 格式：private static 返回值类型 方法名称（参数列表）{方法体}

  • 使用：将上述代码public default void methodCommon改为 private void methodCommon

六、接口中的常量定义及使用

  • 接口当中也可以定义”成员变量“，从效果上看，这其实就是接口的常量。一旦使用final关键字修饰，说明不可改变。

  • 格式：public static final 数据类型 常量名称=数据值；

  • 注意事项：

    1. 接口当中的常量，必须进行赋值。且一旦赋值不可以修改。
    2. 默认public static final三个关键字进行修饰（可以省略，效果不变）。
    3. 不能变的常量名称应使用完全大写（推荐命名规则）。

七、继承父类并实现多个接口

使用接口时需要注意：

  1. 接口中不能有静态代码块。

  2. 接口不能有构造方法。

  3. 一个类的直接父类是唯一的，但是一个类可以同时实现多个接口。

    1. 如果多个接口中含有重名的抽象方法，实现类中覆盖重写一个就行。

    2. 如果不覆盖重写所有接口中的所有抽象方法，那么这个实现类必须是一个抽象类。

    3. 如果实现类所实现的多个接口存在冲突的默认方法，那么实现类中一定要覆盖重写这个默认方法。

    4. 一个类如果直接父类当中的方法和接口当中的默认方法产生了冲突，优先使用父类方法。

八、接口之间的多继承

  • 类与接口之间的关系：
  1. 类与类之间是单继承的，直接父类只有一个。
  2. 类与接口之间是多实现的，一个类可以实现多个接口。
  3. 接口与接口之间是多继承的。

  • 多继承：接口可以继承多个接口。

    • 注意事项：

      1. 多个父接口当中的抽象方法如果重复，无所谓。

      2. 多个接口当中的默认方法重复：必须在子接口必须进行默认方法的覆盖重写。

java学习--数组

发表于 2019-09-12 | 分类于 Java学习笔记

一、数组的定义

  • 定义：数组是一种容器，可以同时存放多个数据值

  • 特点：

    1. 数组是一种引用数据类型。
    2. 数组当中的多个数据，类型必须统一。
    3. 数组的长度在程序运行期间不可改变。

二、数组的初始化

  • 数组的初始化：在内存当中创建一个数组，并且向其中赋予一些默认值。

  • 两种常见的初始化方式：

    1. 动态初始化（指定长度）

      • 数据类型[] 数组名称 = new 数据类型[数组长度];

      • 左侧数据类型：也就是当前数组中保存的数据类型

      • 左侧中括号：代表创建的是数组而不是变量

      • 右侧new：创建数组的动作，为其开辟空间

      • 右侧数据类型：必须与左侧数据类型保持一致

      • 右侧中括号数字：指定数组长度，可以保存多少个同类型数据

      • 注意事项：使用动态初始化数组，元素会拥有一个默认值。

        1. 整数类型：0
          1. 浮点类型：0.0
          2. 字符类型：‘\u0000’
            1. 布尔类型：false
          3. 引用类型：null

  1. 静态初始化（指定内容）

    • 在创建数组的时候不直接指定数据个数，而是直接将具体的数据内容进行指定。

    • 格式：数据类型[] 数组名称 = new 数据类型[] {元素1,元素2,…};

  • 省略格式：数据类型[] 数组名称 = {元素1,元素2,…};

  • 注意事项：

    1. 静态初始化没有直接指定长度，但是仍然会自动推算得到长度。

    2. 静态/静态初始化标准格式都可以拆分成为两个步骤。

    3. 静态初始化一旦使用省略格式则不可拆分。

  1. 使用建议：不确定数组中具体内容时使用静态初始化，否则使用动态初始化。

三、数组的打印

  • 直接打印数组名称，得到的是数组对应的内存地址哈希值（一个16进制的内存地址值）。

  • 访问数组元素：数组名称[索引值]。

    tips: 索引值从0开始，直到“数组长度-1”为止。

四、数组的常见问题

1. 数组索引越界异常

​ 数组索引越界异常(ArrayIndexOutOfBoundsEexception)是指访问数组元素时，索引编号并不存在，就会发生数组索引越界异常。（数组的索引编号从0开始，一直到“数组的长度-1”为止）

2. 空指针异常

​ 所有的引用类型变量，都可以复制为一个null值，代表其中什么都没有。

​ 空指针异常（NullPointerException）:数组必须new初始化才能使用其中的元素，如果数组值赋值为null，而没有初始化创建，那么在使用元素时会发生空指针异常。

五、获取数组的长度

  • 格式：数组长度.length，得到一个int数字即数组长度。

  • 数组一旦创建（new的那个数组），程序运行期间长度不可改变。

七、数组作为方法参数

数组可以作为方法的参数。

当调用方法时，进行传参，传递的参数本质上是数组的地址值。

八、数组作为返回值

一个方法可以有0、1、多个参数；但是只能有一个或0个返回值，不能有多个返回值。

如果希望有多个返回值，可以i用数组作为返回值即可。

任何数据类型都能作为方法的参数类型或者返回值类型。

数组作为方法的参数或返回值，传递的本质上都是数组的地址值。

九、Java内存划分

java内存需要划分成5个部分：

  1. 栈（stack）：存放的都是方法中的局部变量。方法的运行在栈中。

    ​ 局部变量：方法的参数，或者是方法内部的变量。

    ​ 作用域：一旦超出作用域，立刻从栈内存中消失。

  2. 堆（heap）：凡是new出来的东西，都在堆中。

    ​ 堆内存中的东西都有一个16进制的地址值。

    ​ 堆内存中的数据都具有默认值。

  3. 方法区（Method Area）：存储.class相关的信息，包含方法的信息。

  4. 本地方法栈（Native Method Stack）：与操作系统相关。

  5. 寄存器（PC Register）：与CPU相关。

java学习--数组

发表于 2019-09-12 | 分类于 Java学习笔记

一、数组的定义

  • 定义：数组是一种容器，可以同时存放多个数据值
  • 特点：
    1. 数组是一种引用数据类型。
    2. 数组当中的多个数据，类型必须统一。
    3. 数组的长度在程序运行期间不可改变。

二、数组的初始化

  • 数组的初始化：在内存当中创建一个数组，并且向其中赋予一些默认值。

  • 两种常见的初始化方式：

    1. 动态初始化（指定长度）

      • 数据类型[] 数组名称 = new 数据类型[数组长度];

      • 左侧数据类型：也就是当前数组中保存的数据类型

      • 左侧中括号：代表创建的是数组而不是变量

      • 右侧new：创建数组的动作，为其开辟空间

      • 右侧数据类型：必须与左侧数据类型保持一致

      • 右侧中括号数字：指定数组长度，可以保存多少个同类型数据

        ‘ int[] arrayA = new int[300]; ’

  1. 静态初始化（指定内容）

    • 在创建数组的时候不直接指定数据个数，而是直接将具体的数据内容进行指定。

    • 格式：数据类型[] 数组名称 = new 数据类型[] {元素1,元素2,…};

      ‘ int[] arrayB = new int[] {2,15,26}; ’

      ‘ String[] arrayC = new String[]{“hello”,”world”,”java”}; ’

    • 省略格式：数据类型[] 数组名称 = {元素1,元素2,…};

      ‘int[] array A = {10,20,30};‘

    • 注意事项：

      1. 静态初始化没有直接指定长度，但是仍然会自动推算得到长度。

      2. 静态/静态初始化标准格式都可以拆分成为两个步骤。

        ···java

        int[] arrayA ;

        arrayA = new int[]{10,20,30};//静态初始化拆分

        int[] arrayB;

        arrayB = new int[5];//动态初始化拆分

        ···

      3. 静态初始化一旦使用省略格式则不可拆分。

  1. 使用建议：不确定数组中具体内容时使用静态初始化，否则使用动态初始化。

java学习--方法

发表于 2019-09-11 | 分类于 Java学习笔记

一、方法的定义与调用

  • 格式:

    修饰符 返回值类型 方法名（参数列表）{

    ……..//代码实现

    return 结果；

    }

    1. 修饰符：public static
    2. 返回值类型：方法运行结果的数据类型
    3. 参数列表：调用方法时传递所需要的未知数据
    4. return：方法执行后返回结果

  • 方法的三种调用格式：

    1. 单独调用：方法名称（参数）；
    2. 打印调用
    3. 赋值调用

    public class Demo02MethodDefine {

    public static void main(String[] args) {
    sum(10,20);//1.单独调用：方法名称（参数）；
    System.out.println(sum(10,20));//2.打印调用
    int re = sum(10,20);//3.赋值调用
    System.out.println(re);
}
public static int sum(int a,int b){
    int result = a + b;
    return result;
}

    }

  • 定义方法的注意事项：

    1. 定义的位置：在类中，方法外
    2. 返回值类型必须与return语句中返回值类型相同

二、方法重载（overload）

  • 方法重载（定义）：指在同一个类中，使具有同一功能的方法同名，调用时只需要执行相对应参数列表的方法即可，与返回值类型无关。

    tips: windows不区分大小写，代码明确区分大小写，open（）与OPEN()不是重载

  • 参数列表：个数不同，数据类型不同，多类型顺序不同。

  • 重载方法调用：jvm通过方法的参数列表调用不同的方法。

  • 与下列因素无关：

    1. 与参数名称无关
    2. 与方法返回值类型无关

  • 示例：

    public static int sum(int a ,int b){…}

    public static int sum(int a ,int b,int c){…}

    public static int sum(int a ,int b,int c , int d){…}

    public static double sum(double a ,double b){…}

    //在调用输出语句时，println方法也是进行了多种数据类型的重载形式

研究生第一周

发表于 2019-09-11

一、上周生活总结

​ 其实不算是一个一周生活总结，因为阴茎算是到达学校后的第11天了。来之前充满了对新生活的恐惧，好似也没有过多的期待，来之后觉得很多事情没有想象中那么糟糕，比如新舍友、比如我的导师。新的舍友们也都算得上是活泼的性格吧，并不是我想象中的沉闷，也并没有变成我终日一个人踽踽独行的场面，大家还是会一起吃饭，一起去买东西。但是我想，研究生生涯还是要向雷涛院长所说的耐得住寂寞，如大学生活一般长期处于人群之中确实是一种随波逐流的表现。

​ 第一周完成了选课、缴纳学费等工作，到目前为止除了没有购买课本，算是基本上完成了入学的绝大多数工作。在雷涛院长给我们做的入学会议上，他说研究生生涯要给自己定一些目标，有几个至少：至少申请一次出国留学，至少参加一次国际会议，后面忘了，总之都是一些可以去尝试的目标，希望以后有机会能够去完成。

​ 昨天是教师节，和实验室师兄师姐一起去了老师家里送了礼物，本以为就是简单的寒暄就可以离开，结果老师谈了许多关于数学的探讨。其中提到极限的定义，在我们一同思考的时候才发现当初学极限的时候根本没有人过多关注极限的定义，而是将目光都放在计算的方法上，却忘了学习的本质是你需要去认识一个事务，去了解他才能运用它。在那场谈话里，我看得到老师对于学术的热情，对于数学的热爱，我想，也真的只有这样的人适合去做学术研究，因为他们有无穷的好奇心和不断探索的毅力。而我目前所缺失的，正是这种探索的精神，我们总是草草的被动吸收知识，却没有了当初学习很多课程时被新奇事物所吸引的感觉。

二、关于毕业后的目标

​ 当初考研就是为了能够进入大厂，拥有一个光明的前景，不错的薪水。然而昨天去老师家，听他讲毕业师兄的薪水，才恍然大悟，原来我一直所抱有的期待，在这样的氛围里很难实现。和朋友探讨最终发现，原来我们在研究生阶段所需要探索的那些方向，对于我们所向往的公司而言，他们只需要极少的人，而这极少的人很可能就会是更优秀的学校里更优秀的学生，而我们这种普通学校的学生，也许真的只能进入一些所谓的小厂，拿着很一般的薪水，成为他们更具有性价比的选择。到这里我已经很失望了，不知道自己应该如何继续的我研究生生活，感觉当初考研的目标彻底破灭。于是我搜来了一些大厂招聘的要求，希望能够鞭策自己去达到这些目标。
